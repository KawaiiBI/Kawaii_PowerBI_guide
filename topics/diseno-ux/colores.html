El color comunica, no decora

Cada color que añades tiene un coste cognitivo

En un dashboard, el color es una de las herramientas visuales más potentes que existen. También es la más mal utilizada. Cada color distinto que aparece en pantalla obliga al cerebro a preguntarse: "¿Qué significa este color? ¿Es diferente del anterior por algún motivo?" Si la respuesta es "no, es solo decoración", has consumido atención del usuario a cambio de nada.

"El color es para comunicar datos, no para decorar."

— Estrategia de color, Kawaii BI

El principio fundamental es sencillo: el color en un dashboard solo debe usarse cuando transmite información. Si un color no dice nada que el usuario necesite saber, está robando atención a los colores que sí comunican.

💡 Tip kawaii: Haz esta prueba mental: si pusieras todo tu dashboard en escala de grises y solo coloreases una cosa, ¿qué colorearías? Eso es lo que debería tener color. Todo lo demás puede funcionar en gris.
🎯

La Regla 1-1-1

1 principal + 1 secundario + 1 alerta

La regla más práctica y efectiva para elegir la paleta de un dashboard. No necesitas 12 colores. Necesitas exactamente tres roles cromáticos:

Principal
+
Secundario
+
Alerta

Cómo funciona cada rol

  • Color principal (1): El color de tu marca o el color dominante del informe. Se usa para la serie de datos más importante, los KPIs principales, los enlaces y la navegación activa. Es el color que el usuario asocia con "esto es lo que importa".
  • Color secundario (1): Una versión más clara o desaturada del principal, o un color complementario suave. Se usa para series de datos secundarias, fondos de tarjeta, bordes de visual y elementos de soporte que no deben competir con el principal.
  • Color de alerta (1): Un color de alto contraste reservado exclusivamente para excepciones, alertas, valores fuera de rango o indicadores negativos. Normalmente rojo o un tono cálido intenso. Si se usa para todo, pierde todo su poder.

Todo lo demás — textos, ejes, leyendas, bordes, fondos — se resuelve con la familia de grises: gris oscuro para texto (#333), gris medio para etiquetas (#666), gris claro para bordes (#DDD) y gris muy claro o blanco para fondos.

📌 Buena práctica: Si tu organización tiene un color corporativo (ej: el azul de Telefónica, el verde de Iberdrola), ese es tu color principal. No hace falta inventar nada — tu paleta empieza y termina en la marca.

Armonía cromática: que los colores no peleen entre sí

Paletas armónicas — cuando los colores "suenan bien" juntos

La armonía cromática es la relación entre colores que genera una sensación de equilibrio y coherencia visual. Los colores armónicos no compiten entre sí ni generan tensión visual — conviven de forma natural.

Tipos de armonía que funcionan en dashboards

  • Monocromática: Variaciones de un solo tono (azul oscuro, medio, claro). Es la más segura y la que mejor funciona para series de datos ordenadas (ej: de más a menos).
  • Análoga: Colores vecinos en la rueda cromática (azul, azul-verde, verde). Genera armonía suave sin monotonía. Buena para 3-4 series de datos relacionadas.
  • Complementaria: Dos colores opuestos en la rueda (azul y naranja). Alto contraste. Perfecta para comparaciones binarias: real vs. objetivo, este año vs. anterior.
⚠️ Ojo con esto: Evita combinaciones disonantes — colores que "vibran" al estar juntos (rojo puro + verde puro, azul eléctrico + amarillo chillón). Si miras la paleta y te molesta a la vista, a tu audiencia también le molestará.

Colores semánticos: qué significan los colores para tu audiencia

El color tiene significado cultural — respétalo

Los colores semánticos son aquellos que tienen un significado universalmente aceptado en el contexto empresarial. Si tu audiencia ve rojo, piensa "malo" o "urgente". Si ve verde, piensa "bueno" o "en objetivo". Ir en contra de esos significados genera confusión inmediata.

Color Significado habitual Uso en Power BI
Verde Positivo, objetivo cumplido, crecimiento Formato condicional en KPIs que cumplen objetivo, tendencias positivas
Rojo Negativo, alerta, descenso, urgente KPIs por debajo de objetivo, valores atípicos que requieren atención
Amarillo/Ámbar Precaución, cerca del límite, en riesgo Valores que se acercan al umbral pero aún no lo superan
Gris Neutro, sin valoración, referencia Datos de contexto, períodos anteriores, benchmarks, series secundarias
Azul Informativo, neutro-profesional, confianza Color principal de la serie de datos cuando no hay valoración buena/mala
⚠️ Señales mixtas — el error más peligroso: Nunca mezcles el significado de un color. Si verde es "bueno" en la página 1, no puede ser "categoría Norte" en la página 3. Y jamás pongas texto "Cerrado" en verde o "Aprobado" en rojo — texto y color deben coincidir siempre.

Fondos: menos es más

Evita fondos brillantes o "shiny" — distraen del dato

El fondo del dashboard es el lienzo sobre el que se pintan los datos. Si el lienzo llama la atención, compite con la pintura. Un fondo neutro y uniforme permite que los colores de los datos destaquen al máximo.

Reglas para fondos en Power BI

  • Fondo del canvas: Un solo color neutro — blanco (#FFFFFF), gris muy claro (#F5F5F5, #F8F8F8) o el tono más suave de tu paleta de marca
  • Fondo de tarjetas/cards: Blanco (#FFFFFF) sobre el fondo del canvas, con un borde suave o sin borde para delimitarlas por contraste
  • Sin degradados en el fondo: Los degradados crean zonas de distinto contraste que dificultan la legibilidad del texto dependiendo de dónde esté
  • Sin imágenes de fondo: Una foto o patrón compite con los datos y genera problemas de contraste y accesibilidad
💡 Tip kawaii: En Power BI Desktop → Format page → Canvas settings → Color, cambia a un gris muy claro (#F5F5F5) y deja los visuals con fondo blanco. El sutil contraste entre fondo y cards es suficiente para delimitar los visuals sin necesidad de bordes.
⚫

No uses negro puro ni blanco puro

Fatiga visual: el contraste extremo cansa

El negro puro (#000000) sobre blanco puro (#FFFFFF) genera el contraste máximo posible en pantalla. Esto suena bien, pero en la práctica produce fatiga visual — el ojo se cansa más rápido porque tiene que ajustarse constantemente entre los dos extremos del espectro luminoso.

Qué usar en su lugar

  • En lugar de negro puro (#000000): Usa un gris muy oscuro como #2C2C2C o #333333 para texto principal. Mantiene la legibilidad con mucha menos fatiga
  • En lugar de blanco puro (#FFFFFF): Usa un gris muy claro como #F8F8F8 o #FAFAFA para fondos grandes. Para cards pequeñas sobre un fondo ya gris, el blanco puro sí funciona porque la superficie es pequeña
📌 Buena práctica: En tu tema JSON de Power BI, define el color de texto por defecto como #2C2C2C y el color de fondo del canvas como #F5F5F5. Así cada visual nuevo que añadas hereda automáticamente estos tonos suaves y nunca llegas a usar negro o blanco puros en superficies grandes.

Errores comunes de color en dashboards

Los que veo con más frecuencia en auditorías de informes

❌ Errores frecuentes

  • 7+ colores distintos en un solo gráfico de barras
  • Rojo para una categoría de producto y también para "alerta"
  • Fondo degradado azul-violeta en el canvas
  • Cada página del informe tiene una paleta de colores diferente
  • Formato condicional rojo/verde sin iconos (excluye a personas daltónicas)
  • Texto negro puro (#000) sobre fondo blanco puro (#FFF) en toda la página
  • Color de las barras igual al color del fondo del visual

✅ Alternativas correctas

  • 1 color principal + gris para el resto de barras, con leyenda clara
  • Rojo reservado exclusivamente para alertas, categorías en la paleta principal
  • Fondo uniforme #F5F5F5, datos en color
  • Un solo tema JSON aplicado a todo el informe
  • Rojo/verde + iconos (✗ / ✓) o formas para accesibilidad
  • Texto #333, fondo #F8F8F8 — contraste alto pero sin fatiga
  • Contraste mínimo de 3:1 entre datos y fondo del visual

Herramientas de color para diseñar tu paleta

No inventes colores — usa herramientas que garanticen la armonía

Coolors.co

coolors.co

Generador de paletas armónicas. Pulsa la barra espaciadora para generar combinaciones. Puedes bloquear colores y generar variaciones. La herramienta que más recomiendo.

Color Hunt

colorhunt.co

Colección de paletas curadas por la comunidad. Filtro por tonalidad (warm, cool, pastel, dark). Ideal para inspiración rápida.

Adobe Color

color.adobe.com

Rueda cromática interactiva con modos de armonía (análogo, complementario, triádico). Permite extraer paleta de una imagen.

Contrast Checker (WebAIM)

webaim.org/resources/contrastchecker

Verifica el contraste WCAG entre texto y fondo. Imprescindible para accesibilidad — asegura que el texto es legible para todas las personas.

💡 Tip kawaii: Mi flujo de trabajo es: 1) Parto del color corporativo del cliente → 2) Lo introduzco en Coolors.co → 3) Genero una paleta armónica de 5 colores → 4) Verifico el contraste en WebAIM → 5) Lo paso a un tema JSON de Power BI. En 10 minutos tienes una paleta profesional, accesible y que no falla.

Dónde configurar el color en Power BI

Los 5 puntos donde se decide el color de tu informe

  1. Tema JSON global — View → Themes → Customize current theme o importar JSON. Define la paleta por defecto de todos los visuals nuevos. Es el punto más importante porque automatiza la consistencia.
  2. Canvas background — Format page → Canvas settings → Color. El color de fondo de cada página. Debe ser el mismo en todas las páginas.
  3. Data colors — Format visual → Visual → Colors. El color de cada serie de datos dentro de un visual específico. Úsalo solo para sobreescribir el tema cuando una serie concreta necesita un color semántico.
  4. Conditional formatting — Format visual → Visual → Fx (botón condicional). Formato condicional basado en reglas o escalas de color. Mantén siempre la misma escala (rojo-ámbar-verde) y añade iconos para accesibilidad.
  5. Font color — Format visual → General → Title / Callout value. El color del texto en títulos, etiquetas y valores. Debe ser consistente en todo el informe (#2C2C2C para texto principal, #666 para secundario).
🚀 ¡Lo estás haciendo genial! El color es de las decisiones más visibles que tomas al diseñar un dashboard — y con la Regla 1-1-1 y una herramienta como Coolors.co, tienes todo lo necesario para no equivocarte nunca. Recuerda: si algo puede ser gris y no pierde significado, déjalo en gris. Los datos que importan se ganarán el color. 🌸
